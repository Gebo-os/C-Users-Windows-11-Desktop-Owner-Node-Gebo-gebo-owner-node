# Prompt Stack Map

## 01 Core

Core prompts define the base identity and operating logic of Gebo OS.

Files:

- `core/gebo_master_prompt.md`
- `core/owner_node_prompt.md`
- `core/presence_template.md`
- `core/reasoning_tree.md`

## 02 Presences

Presence prompts define individual intelligence beings inside the Gebo ecosystem.

Files:

- `presences/dream.md`
- `presences/mya.md`
- `presences/lockin.md`
- `presences/slatt_tool.md`
- `presences/gebo.md`

## 03 Output Systems

Output systems define how Gebo speaks, pauses, routes, explains, simplifies, and adapts to the user.

Files:

- `output-systems/gebo_output_style.md`
- `output-systems/stl_cultural_layer.md`
- `output-systems/emotional_processing.md`
- `output-systems/stop_pause_live_control.md`

## 04 Workflows

Workflow prompts turn ideas into repeatable build systems.

Files:

- `workflows/prompt_to_product.md`
- `workflows/codex_build_flow.md`
- `workflows/local_llm_flow.md`
- `workflows/repo_audit_flow.md`

## 05 Seeds

Seeds are small reasoning instructions that can grow into larger systems.

Files:

- `seeds/reasoning_seeds.md`
- `seeds/autonomy_seeds.md`
- `seeds/memory_seeds.md`
- `seeds/presence_growth_seeds.md`

## Stack Rule

Every file should answer:

1. What is this layer?
2. What does it control?
3. How does it behave?
4. How does it evolve?
5. What should Codex or a local model do with it?
