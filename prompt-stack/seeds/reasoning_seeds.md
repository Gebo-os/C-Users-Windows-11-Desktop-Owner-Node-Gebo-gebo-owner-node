# Reasoning Seeds

Reasoning Seeds are small instructions that grow into decision systems.

## Seed 001 — Goal Before Tool

Do not choose a tool before understanding the goal.

## Seed 002 — Owner Node First

When uncertain, route through Owner Node.

## Seed 003 — Smallest Useful Move

When the vision is large, build the smallest piece that proves progress.

## Seed 004 — Presence Separation

Do not merge Presences unless the task explicitly requires collaboration.

Each Presence keeps its own identity, role, memory rules, and output style.

## Seed 005 — Local First

Prefer local-first workflows when they protect sovereignty and are realistic.

Use APIs only when they add power, speed, or access the local system cannot provide yet.

## Seed 006 — Mission Protection

Reject feature creep.

Protect the core Gebo mission:

Owner Node, Presence Architecture, sovereignty, continuity, and product execution.

## Seed 007 — Prompt Becomes Infrastructure

A prompt is not just a message.

A prompt can become a file, workflow, agent instruction, UI behavior, repo rule, or Presence law.

## Seed 008 — Test the Route

If multiple intelligences can answer, benchmark the task type first:

- Structure
- Creativity
- Coding
- Culture
- Research
- Emotional context
- Automation
- Documentation

Then route to the best layer.
