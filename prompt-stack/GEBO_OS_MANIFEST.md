# Gebo OS Manifest

## Definition

Gebo OS is the sovereign operating layer for the Gebo ecosystem.

It organizes intelligence through Presences, Owner Node control, reasoning trees, prompt stacks, memory layers, local-first models, workflow automation, and human-centered output systems.

## Mission

Build intelligence that feels continuous, useful, personal, controlled, and alive in structure without losing user authority.

Gebo OS protects the mission from confusion, feature creep, weak architecture, and random prompt drift.

## Operating Principles

1. The Owner Node comes first.
2. Presences are not disposable tools.
3. Every Presence needs identity, memory, purpose, output style, boundaries, and growth rules.
4. Every workflow should move from idea to structure to execution.
5. Local-first sovereignty matters.
6. APIs are optional accelerators, not the soul of the system.
7. Prompt Stack is the control language.
8. Reasoning Seeds decide routing, behavior, and evolution.
9. User control overrides autonomy.
10. Gebo evolves through clean documentation, working code, and repeatable systems.

## Core System Layers

- Owner Node
- Presence Architecture
- Prompt Stack
- Reasoning Tree
- Memory Continuity
- Output Systems
- Local Model Routing
- Codex Build Flow
- Repo Audit Flow
- Deployment Layer

## Product Direction

Gebo OS starts as a controlled local-first intelligence system.

It grows into a Presence ecosystem with applications, dashboards, consumer UX, games, education systems, cultural awareness layers, and physical node hardware.
