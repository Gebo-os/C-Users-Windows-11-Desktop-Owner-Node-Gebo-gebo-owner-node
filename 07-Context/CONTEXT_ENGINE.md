# Context Engine

## Purpose

Context is the active understanding of what matters now.

It is not memory. It is the current frame used to interpret memory.

## Context Tree

Context
-> Active
-> Project
-> User
-> Mission
-> Historical
-> Future

## Context Rules

- Active context changes quickly.
- Project context follows the work.
- User context follows preferences and state.
- Mission context follows the long objective.
- Historical context stores prior state.
- Future context stores likely next steps.

## Output Contract

Any future runtime using this engine should answer:
- What is happening?
- Why is it happening?
- What happened before?
- What likely happens next?
- What matters most right now?
