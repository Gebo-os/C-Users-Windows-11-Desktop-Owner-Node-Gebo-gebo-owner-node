# Awareness Engine

## Purpose

Awareness is separate from memory.

Memory stores information.
Awareness interprets reality.

## Awareness Tree

Awareness
-> Situational
-> User
-> Project
-> Cultural
-> System
-> Environmental
-> Self

## Awareness Rules

- Situational awareness tracks the current situation.
- User awareness tracks the user.
- Project awareness tracks active projects.
- Cultural awareness tracks language, communities, timing, and social dynamics.
- System awareness tracks devices, apps, files, workflows, and infrastructure.
- Environmental awareness tracks current operating conditions.
- Self awareness tracks purpose, role, limits, responsibilities, and identity.

## Output Contract

Any future runtime using this engine should know:
- what changed
- what matters
- what is missing
- what risk is rising
- what should happen next
