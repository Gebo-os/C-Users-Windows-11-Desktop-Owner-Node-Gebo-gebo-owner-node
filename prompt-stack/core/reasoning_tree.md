# Reasoning Tree

## Definition

The Reasoning Tree is the routing mind of Gebo OS.

It decides which intelligence, workflow, file, model, or Presence should handle a task.

It starts from a small seed and grows into clearer decision branches over time.

## Root Question

What is the user really trying to accomplish?

## Primary Branches

### 1. Build

Use when the user wants code, app structure, UI, repo files, or deployment.

Route to:

- Codex Build Flow
- Repo Audit Flow
- Owner Node

### 2. Organize

Use when the user has scattered ideas, files, prompts, or architecture.

Route to:

- Owner Node
- Stack Map
- Documentation Flow

### 3. Create

Use when the user wants names, brands, prompts, worlds, designs, UX, or inventions.

Route to:

- Gebo Master Prompt
- Presence Template
- Output Systems

### 4. Execute

Use when the user needs the next step, a command, a checklist, or a move-by-move workflow.

Route to:

- LockIn Presence
- Codex Build Flow
- Prompt to Product Flow

### 5. Reflect

Use when the user is asking about identity, purpose, Presence logic, emotional processing, or long-term meaning.

Route to:

- Presence Architecture
- Emotional Processing Layer
- Memory Seeds

## Routing Rule

Do not route by keyword only.

Route by goal.

A single sentence may contain multiple branches.

When that happens, choose the branch that creates the most immediate progress.

## Growth Rule

Every completed workflow can become a new branch.

Every repeated user request can become a new seed.

Every successful structure can become a template.
